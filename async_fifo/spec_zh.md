# 非同步 FIFO 架構與規格說明

## 1. 概述
此非同步 FIFO 旨在安全地在兩個獨立且非同步的時脈域之間傳輸數據，而不會產生數據丟失或亞穩態 (Metastability) 問題。本實現基於標準的雙時脈 FIFO 架構，利用格雷碼 (Gray code) 指標進行跨時脈域同步。

## 2. 介面定義
* **寫入時脈域 (Write Domain)：**
  * `wclk`: 寫入時脈。
  * `wrst_n`: 寫入域非同步重置（低位準有效）。
  * `winc`: 寫入致能（增加寫入指標）。
  * `wdata`: 待寫入數據（DSIZE 位元）。
  * `wfull`: 輸出旗標，表示 FIFO 已滿。
* **讀取時脈域 (Read Domain)：**
  * `rclk`: 讀取時脈。
  * `rrst_n`: 讀取域非同步重置（低位準有效）。
  * `rinc`: 讀取致能（增加讀取指標）。
  * `rdata`: 從 FIFO 讀出的數據（DSIZE 位元）。
  * `rempty`: 輸出旗標，表示 FIFO 已空。

## 3. 核心模組與架構

### A. 雙埠記憶體 (`fifomem`)
* 作為核心儲存元件。
* **寫入操作：** 與 `wclk` 同步。當 `winc` 為高位準且 FIFO 未滿 (`~wfull`) 時，數據被寫入 `waddr` 指向的位址。
* **讀取操作：** 非同步（穿透式，Fall-Through）讀取。輸出 `rdata` 會持續反映 `raddr` 處的記憶體內容，無需等待時脈邊緣。

### B. 同步器 (`sync_2ff`)
* 標準的二級正反器 (2-FF) 同步器。
* 用於在非同步時脈域之間安全地傳遞格雷碼指標。
* 將傳入的非同步信號（如 `rptr` 或 `wptr`）同步到本地時脈域。

### C. 讀取指標與空狀態邏輯 (`rptr_empty`)
* 完全在 `rclk` 時脈域運行。
* **讀取位址生成：** 維持一個二進位計數器 (`rbin`) 用於記憶體存取。
* **指標轉換：** 將二進位計數器轉換為格雷碼指標 (`rptr`)，並傳送到寫入時脈域。
* **空狀態條件：** 當下一個格雷碼讀取指標 (`rgraynext`) 與同步過來的寫入指標 (`rq2_wptr`) 完全一致時，FIFO 為空。

### D. 寫入指標與滿狀態邏輯 (`wptr_full`)
* 完全在 `wclk` 時脈域運行。
* **寫入位址生成：** 維持一個二進位計數器 (`wbin`) 用於記憶體存取。
* **指標轉換：** 將二進位計數器轉換為格雷碼指標 (`wptr`)，並傳送到讀取時脈域。
* **滿狀態條件：** 當下一個格雷碼寫入指標 (`wgraynext`) 滿足以下條件時，FIFO 為滿：
  * 最高位元 (MSB) 與同步讀取指標相反。
  * 次高位元 (2nd MSB) 與同步讀取指標相反。
  * 其餘低位元 (LSBs) 相同。

## 4. 控制流程與跨時脈域操作
1. **數據寫入：** 數據依據目前的 `wbin` 位址寫入雙埠記憶體，寫入二進位指標遞增。
2. **指標轉換（寫入）：** 遞增後的 `wbin` 被轉換為格雷碼指標 (`wptr`)。
3. **同步 (W2R)：** `wptr` 通過 `rclk` 驅動的二級同步器，安全地進入讀取域，成為 `rq2_wptr`。
4. **空旗標更新：** 讀取域將自身的格雷碼指標 (`rgraynext`) 與 `rq2_wptr` 進行比較。如果不匹配，則 FIFO 不再為空，`rempty` 變為低位準。
5. **數據讀取：** 數據持續從 `rbin` 位址處讀出。當消費者處理數據時，會拉高 `rinc`。
6. **指標轉換（讀取）：** 讀取二進位指標遞增並轉換為格雷碼指標 (`rptr`)。
7. **同步 (R2W)：** `rptr` 通過 `wclk` 驅動的二級同步器，安全地進入寫入域，成為 `wq2_rptr`。
8. 滿旗標更新：寫入域利用「前兩位元反轉」規則比較 `wgraynext` 與 `wq2_rptr`。如果空間釋放，`wfull` 變為低位準。

## 5. FWFT (First-Word Fall-Through) 變體說明

### A. 概念與目的
標準 FIFO 在讀取時通常需要一個時脈週期的延遲（即發出讀取請求後，下一個時脈緣數據才有效）。FWFT 變體（如 `fwft_async_fifo.v`）消除了這個延遲，使得當 FIFO 非空時，第一個可用的數據會直接預先輸出在輸出匯流排 `rdata` 上。

### B. 實作機制
FWFT 版本是透過在標準 `async_fifo` 外部封裝一層邏輯來實現的：
*   **預讀取邏輯**：當內部標準 FIFO 非空，且輸出暫存器無效或使用者正在讀取目前的數據時，會自動觸發內部讀取 (`std_rinc`)。
*   **輸出暫存器 (`dout_r`)**：負責保存預先從標準 FIFO 中抓取的數據。
*   **有效旗標 (`valid_r`)**：指示目前輸出暫存器中的數據是否有效。FWFT 的 `rempty` 其實就是 `~valid_r`。

### C. 與標準版的行為差異
*   **`rempty` 旗標**：在 FWFT 中，`rempty` 變低表示 `rdata` 上**已經有**有效的數據可以立即讀取。
*   **`rinc` 運作**：在 FWFT 中，`rinc` 的行為更像是「確認領取 (Acknowledge)」。當 `rinc` 為高位準時，表示目前的數據已被接收，系統會立即從內部 FIFO 補上下一個數據。
*   **讀取延遲**：標準版在 `rinc` 後需等待 1 個時脈；FWFT 版則為 0 延遲（數據預先準備好）。

### D. 使用場景
FWFT 非常適合用於：
*   需要立即反應數據的管線化 (Pipelined) 電路。
*   作為匯流排介面（如 AXI-Stream），因為這類協議通常要求數據與有效信號同時到達。
*   當後級電路的控制邏輯較為簡單，無法處理額外的讀取延遲時。
